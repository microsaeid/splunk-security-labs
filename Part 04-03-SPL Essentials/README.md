## Search Command

The following search was used to display events generated by the Windows Server 2022 domain controller:

```spl
host=DC01
```

The results confirmed that events associated with the `DC01` host were available in Splunk.

![Search by Host](screenshots/01-search-host-dc01.png)

## Table Command

The `table` command was used to display only the selected fields in a structured table:

```spl
host=DC01
| table host sourcetype source
```

The search results were displayed using the following columns:

- `host`
- `sourcetype`
- `source`

![Table Command](screenshots/02-table-command.png)

## Fields Command

The `fields` command was used with the minus sign to remove the `source` field from the search pipeline:

```spl
*
| fields - source
```

The `source` field was removed from the selected fields while the original raw event text remained unchanged.

![Fields Exclude](screenshots/04-fields-exclude.png)

## Sort Command

### Ascending Sort

The following search was used to sort events by the `host` field in ascending alphabetical order:

```spl
*
| sort host
```

![Ascending Sort](screenshots/05-sort-ascending.png)

### Descending Sort

The minus sign was used before the field name to reverse the sorting order:

```spl
*
| sort - host
```

This arranged the host values in descending alphabetical order.

![Descending Sort](screenshots/06-sort-descending.png)

### Sort by Time

The following search was used to sort events by the `_time` field in descending order:

```spl
*
| sort - _time
```

This placed the newest events at the beginning of the results.

![Sort by Time](screenshots/07-sort-by-time.png)

## Head Command

The `head` command was used to return only the first five events from the search results:

```spl
*
| head 5
```

The search returned five events from the beginning of the result set.

![Head Command](screenshots/08-head-5-events(1).png)

## Tail Command

The `tail` command was used to return only the last five events from the search results:

```spl
*
| tail 5
```

The search returned five events from the end of the result set.

![Tail Command](screenshots/09-tail-5-events.png)

## Key Observations

- The `search` command filters events based on specified conditions.
- The pipe symbol (`|`) passes the results of one command to the next command.
- The `table` command displays selected fields in a structured table.
- The `fields -` command removes specified fields from the search pipeline.
- The `sort` command organizes results by a selected field.
- A minus sign before a sort field applies descending order.
- The `head` command returns the first specified number of results.
- The `tail` command returns the last specified number of results.

## Conclusion

Essential SPL commands were successfully used to search, format, remove fields, sort events, and limit the number of displayed Windows Event Log results in Splunk.
