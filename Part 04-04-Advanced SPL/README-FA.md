# Part 04-04 — دستورات پیشرفته SPL

## معرفی

### جدول خلاصه

| مورد | جزئیات |
|------|---------|
| پروژه | دستورات پیشرفته SPL |
| برنامه | Search & Reporting |
| بازه زمانی جستجو | 60 دقیقه گذشته |
| منبع داده | Windows Event Logs |
| میزبان | DC01 |
| دستورات SPL | dedup، rename، where |
| هدف | استفاده از دستورات پیشرفته SPL برای حذف نتایج تکراری، تغییر نام فیلدها و فیلترکردن Eventهای امنیتی ویندوز |

## محیط لاب

جستجوهای این لاب در برنامه Search & Reporting در Splunk Enterprise انجام شدند.

Windows Event Logهای استفاده‌شده در این لاب از Windows Server 2022 Domain Controller با نام میزبان زیر جمع‌آوری شده بودند:

`DC01`

بازه زمانی انتخاب‌شده برای جستجو نیز به شکل زیر بود:

`Last 60 minutes`

## مراحل لاب

### 1. حذف نتایج تکراری با دستور dedup

جستجوی زیر اجرا شد:

```spl
host=dc01
| dedup host
```

در مرحله اول، جستجوی `host=dc01` تمام Eventهای مربوط به میزبان `DC01` را در بازه زمانی انتخاب‌شده نمایش داد.

از آنجا که همه Eventهای بازگشتی دارای مقدار یکسان `DC01` در فیلد `host` بودند، دستور `dedup host` نتایج تکراری را بر اساس این فیلد حذف کرد.

در نتیجه، Splunk فقط یک Event را به‌عنوان نماینده مقدار منحصربه‌فرد `DC01` نگه داشت.

پس از اجرای این دستور، تعداد نتایج به یک Event کاهش پیدا کرد.

![حذف نتایج تکراری بر اساس Host](screenshots/1-dedup.png)

### 2. تغییر نام فیلد با دستور rename

جستجوی زیر اجرا شد:

```spl
host=dc01
| rename host AS computer
```

دستور `rename` نام فیلد `host` را در نتایج جستجو به `computer` تغییر داد.

پس از اجرای جستجو، فیلد جدید `computer` در بخش فیلدهای موجود نمایش داده شد.

این تغییر فقط روی نتایج همان جستجو اعمال شد و نام اصلی فیلد `host` را به‌صورت دائمی در Splunk تغییر نداد.

در زمان اجرای این جستجو، تعداد 550 Event مربوط به میزبان `DC01` نمایش داده شد.

![تغییر نام فیلد Host](screenshots/2-Rename.png)

### 3. فیلترکردن Eventها با دستور where

جستجوی زیر اجرا شد:

```spl
host=dc01
| where EventCode=4624
```

در مرحله اول، جستجو Eventهای مربوط به میزبان `DC01` را انتخاب کرد.

سپس دستور `where` مقدار فیلد `EventCode` را در هر Event بررسی کرد و فقط Eventهایی را نگه داشت که مقدار آنها برابر با `4624` بود.

Event ID 4624 در Windows Security Log نشان‌دهنده ورود موفق یک حساب کاربری است.

پس از اجرای این شرط، تعداد 180 Event در بازه زمانی 60 دقیقه گذشته نمایش داده شد.

نتایج همچنین نشان دادند که این Eventها از Windows Security Log با sourcetype زیر دریافت شده‌اند:

`WinEventLog:Security`

![فیلترکردن Event ID 4624](screenshots/3-Where.png)

## نتایج

هر سه دستور پیشرفته SPL با موفقیت روی Windows Event Logهای دریافت‌شده از `DC01` آزمایش شدند.

دستور `dedup` نتایج تکراری مربوط به فیلد `host` را حذف کرد و فقط یک نتیجه منحصربه‌فرد را نگه داشت.

دستور `rename` نام نمایشی فیلد `host` را به `computer` تغییر داد.

دستور `where` فقط Eventهای دارای Event ID 4624 را از میان Eventهای Windows Security نمایش داد.

## دستورات استفاده‌شده

```spl
host=dc01
| dedup host
```

```spl
host=dc01
| rename host AS computer
```

```spl
host=dc01
| where EventCode=4624
```

## جمع‌بندی

در این لاب، سه دستور پیشرفته SPL در محیط Splunk Enterprise آزمایش شدند.

دستور `dedup` برای حذف نتایج تکراری بر اساس یک فیلد مشخص استفاده شد.

دستور `rename` برای تغییر نام نمایشی یک فیلد و خواناترکردن نتایج جستجو مورد استفاده قرار گرفت.

دستور `where` برای اعمال یک شرط روی فیلد `EventCode` استفاده شد و فقط Eventهای مربوط به ورود موفق ویندوز با Event ID 4624 نمایش داده شدند.

هر سه جستجو با موفقیت اجرا شدند و نتایج مورد انتظار در Splunk Enterprise مشاهده و تأیید شدند.
