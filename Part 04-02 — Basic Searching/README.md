## Basic Search

The following search was used to display all available events within the selected time range:

```spl
*
```

The search returned Windows events from both monitored systems.

![Basic Search](https://github.com/microsaeid/splunk-security-labs/blob/main/Part%2004-02%20%E2%80%94%20Basic%20Searching/1-Basic-Search.png)

## Search by Host

The following search was used to display events generated by the Windows Server 2022 domain controller:

```spl
host=DC01
```

The results confirmed that events associated with the `DC01` host were available in Splunk.

![Search by Host](screenshots/2-Search-by-Host.png)

## Search by Sourcetype

The following search was used to display Windows Application events:

```spl
sourcetype="WinEventLog:Application"
```

The results contained only events from the Windows Application log.

![Search by Sourcetype](screenshots/3-Search-by-Sourcetype.png)

## Combined Search

Multiple fields were combined to narrow the results to Security events generated by `DC01`:

```spl
host=DC01 source="WinEventLog:Security"
```

The search returned only Windows Security events associated with the domain controller.

![Combined Search](screenshots/4-Mix-Search.png)

## Selected Fields

The `Selected Fields` panel was used to identify the actual values available in the indexed events.

The following fields were reviewed:

- `host`
- `source`
- `sourcetype`

The available host values included:

```text
DC01
DESKTOP-S8HR81A
```

The available Windows Event Log values included:

```text
WinEventLog:Application
WinEventLog:Security
WinEventLog:System
```

## Verification

The searches confirmed that:

- Windows events were searchable in Splunk.
- Events could be filtered by host.
- Events could be filtered by source and sourcetype.
- Multiple search conditions could be combined.
- The actual indexed field values could be identified through the Selected Fields panel.
