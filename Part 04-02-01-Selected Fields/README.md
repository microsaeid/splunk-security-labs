# Part 04-02-01 — Selected Fields

## Overview

This lab demonstrates how to use the **Selected Fields** panel in Splunk Search & Reporting to explore available field values and filter search results.

## Lab Environment

- Splunk Enterprise 10.0.2
- Ubuntu Server 24.04.4 LTS

## Steps

### 1. Execute a Basic Search

Executed the following search:

```spl
*
```

Verified that the **Selected Fields** panel displayed the available fields.

![Selected Fields](https://github.com/microsaeid/splunk-security-labs/blob/main/Part%2004-02-01-Selected%20Fields/1-selected-fields.png)

---

### 2. Review Host Values

Selected the **host** field to display the available host values returned by the search.

![Host Values](https://github.com/microsaeid/splunk-security-labs/blob/main/Part%2004-02-01-Selected%20Fields/2-selected-field-host-values.png)

---

### 3. Filter Search Results

Selected the `DC01` host value to automatically update the search and display only events generated by the selected host.

![Filter by Host](https://github.com/microsaeid/splunk-security-labs/blob/main/Part%2004-02-01-Selected%20Fields/3-filter-by-host-from-selected-fields.png)

---

### 4. Review Event Details

Opened an event and verified that the selected fields were displayed in the event details.

![Event Details](https://github.com/microsaeid/splunk-security-labs/blob/main/Part%2004-02-01-Selected%20Fields/4-event-fields.png)

## Result

Successfully explored the available field values using the **Selected Fields** panel and filtered Windows events directly from the search interface.
